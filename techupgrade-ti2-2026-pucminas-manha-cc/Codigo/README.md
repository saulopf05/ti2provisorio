# Código do Projeto

O código do projeto foi desenvolvido utilizando a IDE Eclipse, que possui uma estrutura própria e padronizada de organização dos arquivos. Essa estrutura facilita a separação entre o código-fonte e os arquivos compilados, contribuindo para uma melhor organização e manutenção do sistema. No Eclipse, o diretório src é utilizado para armazenar todo o código-fonte do projeto, incluindo as classes principais e demais componentes da aplicação. Já o diretório bin contém os arquivos compilados automaticamente pela IDE, ou seja, os arquivos .class, que são gerados a partir do código escrito em Java. Dependendo da configuração do projeto, também podem existir pastas adicionais, como lib, destinada a bibliotecas externas, e resources, utilizada para armazenar arquivos auxiliares, como dados em formato JSON, imagens ou outros recursos necessários para a execução do sistema.

Além disso, dentro da pasta src, o projeto pode estar organizado em pacotes, o que permite separar melhor as responsabilidades de cada parte do sistema, como classes de controle, modelos de dados e interface. Essa organização segue boas práticas de desenvolvimento e torna o código mais legível, reutilizável e fácil de manter. Dessa forma, o uso da estrutura padrão do Eclipse não apenas atende aos requisitos do projeto, mas também contribui para um desenvolvimento mais organizado e profissional.
